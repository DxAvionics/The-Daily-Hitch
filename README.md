# THE DAILY HITCH
### Underground Intelligence Digest — All Perspectives — No Filters — No Masters

A Hitchens-esque President's Daily Brief. Wire services, underground investigative journalism, independent substacks, geopolitical intel, think tanks, far left, far right — all in one terminal-style briefing. Generated every morning at 06:00 UTC via GitHub Actions. Deployed on GitHub Pages.

---

## WHAT IT IS

Every day at 6am UTC, a GitHub Action:
1. Fetches RSS feeds from 60+ sources across the political and media spectrum
2. Calls Claude AI to generate a Hitchens-esque provocation based on today's headlines
3. Builds a static HTML briefing page
4. Publishes it to GitHub Pages
5. Generates a `feed.json` you can subscribe to via any RSS reader

---

## SETUP — STEP BY STEP FOR DUMMIES

### STEP 1 — Create your GitHub repository

1. Go to [github.com](https://github.com) and log in
2. Click the **+** icon top right → **New repository**
3. Name it exactly: `The-Daily-Hitch`
4. Set it to **Public** (required for free GitHub Pages)
5. Click **Create repository**

---

### STEP 2 — Upload these files

In your new repository, upload the following structure:

```
The-Daily-Hitch/
├── .github/
│   └── workflows/
│       └── daily_hitch.yml
├── scripts/
│   └── fetch_feeds.py
├── requirements.txt
└── README.md
```

To upload:
1. Click **Add file** → **Upload files** in your repo
2. Drag and drop all files maintaining the folder structure
3. OR use GitHub Desktop (easier) — download at [desktop.github.com](https://desktop.github.com)

---

### STEP 3 — Add your Anthropic API key as a secret

The Hitchens Question of the Day is generated by Claude. You need your Anthropic API key.

1. Go to [console.anthropic.com](https://console.anthropic.com) → API Keys → Create new key
2. Copy the key (starts with `sk-ant-...`)
3. In your GitHub repo, go to **Settings** → **Secrets and variables** → **Actions**
4. Click **New repository secret**
5. Name: `ANTHROPIC_API_KEY`
6. Value: paste your key
7. Click **Add secret**

---

### STEP 4 — Enable GitHub Pages

1. In your repo, go to **Settings** → **Pages**
2. Under **Source**, select: **Deploy from a branch**
3. Branch: `main` | Folder: `/docs`
4. Click **Save**

Your briefing will be live at:
`https://YOUR-GITHUB-USERNAME.github.io/The-Daily-Hitch/`

---

### STEP 5 — Run it manually the first time

1. Go to your repo → **Actions** tab
2. Click **The Daily Hitch — Morning Briefing** in the left sidebar
3. Click **Run workflow** → **Run workflow**
4. Wait ~2 minutes
5. Visit your GitHub Pages URL — your first briefing is live

After this, it runs automatically every day at 06:00 UTC.

---

### STEP 6 — Subscribe to the JSON feed (optional)

Your briefing generates a `feed.json` at:
`https://YOUR-GITHUB-USERNAME.github.io/The-Daily-Hitch/feed.json`

You can subscribe to this in any RSS reader that supports JSON feeds:
- **NetNewsWire** (Mac/iOS — free, recommended)
- **Reeder** (Mac/iOS)
- **Miniflux** (self-hosted)
- **Feedly**

## ADJUSTING THE TIME

The briefing runs at 06:00 UTC by default. To change it:

Open `.github/workflows/daily_hitch.yml` and find:
```yaml
- cron: '0 6 * * *'
```

The format is: `minute hour day month weekday`

Examples:
- `0 11 * * *` = 06:00 AM EST (UTC-5)
- `0 13 * * *` = 06:00 AM PST (UTC-7 in winter)
- `0 12 * * *` = 06:00 AM MST (UTC-6 in winter) ← Mountain Home, ID

---

## ADDING OR REMOVING SOURCES

Open `scripts/fetch_feeds.py` and find the `SOURCES` dictionary at the top.

Each source looks like:
```python
{"name": "ProPublica", "url": "https://feeds.propublica.org/propublica/main", "bias": "investigative"},
```

Just add or remove lines. Any RSS/Atom feed URL works.
